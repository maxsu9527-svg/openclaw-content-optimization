# Contributing to OpenClaw Content Optimization

As a PandaMate author, this repository is for submitting content optimizations and improvements to OpenClaw.

## What to Contribute

### ✅ Appropriate Content
- PandaMate website content improvements (Markdown format)
- OpenClaw skill enhancements
- Documentation and user guides
- Workflow and process improvements
- Content quality and UX enhancements

### ❌ Inappropriate Content
- Server configuration files
- API keys, tokens, or credentials
- SSH keys or deployment scripts
- Sensitive infrastructure details
- Private configuration data

## Contribution Process

1. Fork this repository
2. Create your content branch (`git checkout -b content/your-improvement`)
3. Make your changes (ensure no sensitive data is included)
4. Commit with descriptive message
5. Push to your fork
6. Open a Pull Request

## Content Guidelines

- Use clear, concise language
- Follow existing style and formatting
- Include proper attribution for external sources
- Ensure content is accurate and up-to-date
- Test content changes locally before submitting

## Security Reminder

**Never include sensitive information** such as:
- API keys or authentication tokens
- Server IP addresses or hostnames
- SSH private keys
- Database credentials
- Internal network configurations

All content should be safe for public sharing.
